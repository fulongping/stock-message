# 投资主题与微信观点信号 Web 工具

一个零依赖的 Node Web 小工具，当前包含三部分能力：

- Part 1：每 10 分钟抓取一次同花顺首页，从“主题投资”区块提取 3 个今日最热主题；区间涨幅榜只统计 `60` 和 `00` 打头的沪深主板股票，并且仅在 `9:30-11:30`、`13:00-15:00` 交易时段内按 5 分钟整点运行。每个 5 分钟整点都会保存一份当日区间前十历史；遇到 `:00 / :15 / :30 / :45` 这些 15 分钟节点，主榜会切换为过去 15 分钟的前十，其余节点展示当前 5 分钟前十。
- Part 2：交易日 `15:05` 之后，自动抓取同花顺首页前 5 大热门主题，映射到同花顺概念题材页，拉取题材成分股和日线数据，从中筛出符合你模式的 5 只股票。`模式` 当前被定义为：底部起来放量至少三天、基本沿 `MA5` 上行、最近没有连续两天跌破 `MA5`。`00` / `60` 之外的股票不会进入筛选池。
- Part 3：每 10 分钟扫描一次本地导入目录，读取微信群“子明和他的朋友们”和公众号“子明解读”的可读文本，重点整理顾子明、子明群-子路、间歇泉三人的发言，并判断当前讨论更偏“科技向”还是“战争线”。

## 启动

```bash
node server.js
```

默认地址：

```text
http://127.0.0.1:3000
```

如果环境里能正常执行 `npm`，也可以用：

```bash
npm start
```

## 同花顺区间涨幅前十的工作方式

同花顺增强部分现在会做两件事：

- 抓首页“主题投资”区块，展示 3 个今日最热主题。
- 同时从同花顺移动排行接口拉取 `涨跌幅降序前 3000` 和 `涨跌幅升序前 3000` 两份快照，合并去重后只保留 `60` / `00` 打头的沪深主板股票，得到更完整的市场样本。

随后系统会在每个 5 分钟整点把“当前快照价格”和“对应基准快照价格”做对比，得到：

- 当前主榜的区间涨幅前 10 股票。
- 当日全部 5 分钟区间的前 10 历史记录。
- 每只股票的区间涨幅、当前价格、当日涨跌幅、换手率、成交量。
- 每只股票对应的同花顺概念题材。
- 概念分类聚合结果。

说明：

- 只统计 `60` / `00` 打头的沪深主板股票，科创板、创业板、北交所都不会纳入样本。
- 自动统计只在 `9:30-11:30`、`13:00-15:00` 进行，并且按 5 分钟整点对齐。
- 每个 5 分钟整点都会保存一份当日历史榜单。
- 到了 `:00 / :15 / :30 / :45` 这些 15 分钟节点，主榜会切换为过去 15 分钟的前十；其他整点仍显示当前 5 分钟前十。
- 首轮抓取只能建立基准快照，等到下一轮有效整点快照到来后才会出现区间涨幅前十。


## 收盘复盘与模式筛选

收盘复盘模块会在交易日 `15:05` 后启动，并做下面几步：

- 从同花顺首页继续提取前 5 个热门主题。
- 把主题名称映射到同花顺概念题材页，收集对应成分股。
- 只保留 `00` / `60` 开头股票，并优先评估这些热门主题里当日更活跃的一批候选。
- 对每只候选股票抓取同花顺日线 `all.js` 数据，按同花顺官方 `d3_charts.hq.js` 的压缩解码逻辑还原前复权日 K。
- 计算 `MA5`、最近是否连续跌破 `MA5`、以及最近一段是否存在“底部起来放量至少三天”的连升区间。

当前“底部起来放量至少三天”的工程化定义是：

- 最近 12 根日 K 内，存在一段长度至少 3 天的连续收盘上涨区间。
- 同一段区间内，成交量整体递增，且至少出现 2 次明显放量。
- 这段上涨起点的最低价，接近此前 13 根日 K 的局部低点。

这是一个偏实战化的近似定义，目的是把题材热度和你给出的走势约束组合起来，而不是机械复刻通达信公式。

如果当天严格满足条件的股票不足 5 只，页面会继续从同主题股票里补齐最接近模式的候选，并在警告区明确标记“补位”。
## 微信信号的工作方式

微信分析默认从下面这个目录读取导入文件：

```text
D:\message\data\wechat-import
```

支持的文件类型：

- `json`
- `jsonl`
- `ndjson`
- `txt`
- `md`
- `html`

系统会优先提取目标群聊和目标公众号的内容，并从最近 100 条群消息里重点识别：

- `顾子明`
- `子明群-子路`
- `间歇泉`

分析结果会输出：

- 当前判断：`偏科技向` / `偏战争线` / `拉锯中` / `信号不足`
- 科技线与战争线关键词得分
- 最近群消息摘录
- 最近公众号文章摘要
- 三位重点发言人的单独卡片

## 导入格式示例

参考 [examples/wechat-import-sample.json](/D:/message/examples/wechat-import-sample.json)。

示例结构：

```json
[
  {
    "chatName": "子明和他的朋友们",
    "sourceName": "子明和他的朋友们",
    "messages": [
      {
        "timestamp": "2026-03-24T09:40:00+08:00",
        "sender": "顾子明",
        "content": "今天盘面更愿意交易算力、服务器和国产替代，主线还是科技扩散。"
      }
    ]
  },
  {
    "sourceName": "子明解读",
    "articles": [
      {
        "timestamp": "2026-03-24T08:00:00+08:00",
        "title": "科技扩散与事件冲击之间",
        "content": "如果中东没有继续升级，市场主线仍在算力、芯片、机器人和国产替代。"
      }
    ]
  }
]
```

## 当前限制

当前这台机器已经检测到新版微信桌面端本地存储，但消息数据库不是明文 SQLite。也就是说：

- 这个版本不会直接解密读取微信实时消息库。
- 当前稳定方案是分析导入到 `data/wechat-import` 的可读文本或 JSON 文件。
- 只要导入文件更新，服务端就会在下一次 10 分钟轮询时自动重新分析。

## 说明

- 同花顺抓取逻辑依赖首页当前的 SSR 结构。
- 如果同花顺调整 DOM 或反爬策略，需要同步调整 [src/hotThemesService.js](/D:/message/src/hotThemesService.js)。
- 区间涨幅前十逻辑位于 [marketLeadersService.js](/D:/message/marketLeadersService.js)。
- 如果后续拿到稳定的微信导出桥接程序，可以继续把第二部分升级成真正的自动同步。
