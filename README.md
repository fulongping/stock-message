# 投资主题与微信观点信号 Web 工具

一个零依赖的 Node Web 小工具，当前包含三部分能力：

- Part 1：每 10 分钟抓取一次同花顺首页，从“主题投资”区块提取 3 个今日最热主题；区间涨幅榜只统计 `60` 和 `00` 打头的沪深主板股票，并且统一剔除 `ST / *ST`，仅在 `9:30-11:30`、`13:00-15:00` 交易时段内按 5 分钟整点运行。每个 5 分钟整点都会保存一份当日区间前十历史；遇到 `:00 / :15 / :30 / :45` 这些 15 分钟节点，主榜会切换为过去 15 分钟的前十，其余节点展示当前 5 分钟前十。页面还会额外固定展示当前 5 分钟回落最快的 5 只股票。
- Part 2：交易日 `15:05` 之后，自动抓取同花顺首页前 5 大热门主题，映射到同花顺概念题材页，拉取题材成分股和日线数据，从中筛出符合你模式的 5 只股票。当前模式已经进一步收紧为“近 6 日量能相对近 10 日刚抬升 + 沿 MA5 稳定推进 + 乖离不过大”的趋势推进型审美，用来避开量能铺得太久的偏后排走势。页面还会同步回放最近 10 个信号日，统计“T+1 开盘买、T+2 收盘卖”的盈亏表现。
- Part 3：每 10 分钟扫描一次本地导入目录，读取微信群“子明和他的朋友们”和公众号“子明解读”的可读文本，重点整理顾子明、子明群-子路、间歇泉三人的发言，并判断当前讨论更偏“科技向”还是“战争线”。

## 启动

```bash
node server.js
```

默认地址：

```text
http://127.0.0.1:3000
```

如果环境里能正常执行 `npm`，也可以用：

```bash
npm start
```

## 同花顺区间涨幅前十的工作方式

同花顺增强部分现在会做两件事：

- 抓首页“主题投资”区块，展示 3 个今日最热主题。
- 同时从同花顺移动排行接口拉取 `涨跌幅降序前 3000` 和 `涨跌幅升序前 3000` 两份快照，合并去重后只保留 `60` / `00` 打头的沪深主板股票，得到更完整的市场样本。

随后系统会在每个 5 分钟整点把“当前快照价格”和“对应基准快照价格”做对比，得到：

- 当前主榜的区间涨幅前 10 股票。
- 当日全部 5 分钟区间的前 10 历史记录。
- 当前 5 分钟窗口里回落最快的 5 只股票。
- 每只股票的区间涨幅、当前价格、当日涨跌幅、换手率、成交量。
- 每只股票对应的同花顺概念题材。
- 概念分类聚合结果。

说明：

- 只统计 `60` / `00` 打头的沪深主板股票，科创板、创业板、北交所都不会纳入样本；`ST / *ST` 也会统一排除。
- 自动统计只在 `9:30-11:30`、`13:00-15:00` 进行，并且按 5 分钟整点对齐。
- 每个 5 分钟整点都会保存一份当日历史榜单。
- 到了 `:00 / :15 / :30 / :45` 这些 15 分钟节点，主榜会切换为过去 15 分钟的前十；其他整点仍显示当前 5 分钟前十。
- 首轮抓取只能建立基准快照，等到下一轮有效整点快照到来后才会出现区间涨幅前十和当前 5 分钟回落榜。

## 收盘复盘与模式筛选

收盘复盘模块会在交易日 `15:05` 后启动，并做下面几步：

- 从同花顺首页继续提取前 5 个热门主题。
- 把主题名称映射到同花顺概念题材页，收集对应成分股。
- 只保留 `00` / `60` 开头且非 `ST / *ST` 的股票，并优先评估这些热门主题里当日更活跃的一批候选。
- 对每只候选股票抓取同花顺日线 `all.js` 数据，按同花顺官方 `d3_charts.hq.js` 的压缩解码逻辑还原前复权日 K。
- 用趋势推进型规则评估候选，而不是再硬性要求“连续三天爆量”。

当前工程化定义是：

- 最近 `10` 天里，大多数交易日都贴着 `MA5` 运行，并且最近没有连续 `2` 天跌破 `MA5`。
- `MA5` 继续向上，但斜率不能太陡，尽量保留“顺着 5 日线推”的形态。
- 收盘价相对 `MA5` 的乖离率要舒服，优先保留大约 `-1%` 到 `8.5%` 的票；但对量能久铺、沿 MA5 稳定推进的票，会单独放宽到约 `10%`。
- 量能看中枢抬升而不是三天连量：重点看 `10` 日均量是否高于 `20` 日均量，以及最近 `10` 天里有多少天成交量高于 `20` 日均量。
- 走势上要求已经脱离近 `20` 日低点，但又不能离底部和 `MA5` 太远，尽量保留“趋势推进前半段”的图形；稳态推进股会允许更宽一点的底部抬升空间。
- 对那种量能已经铺了更久、MA5 几乎天天踩稳的票，允许近 `10` 天里多出 `1` 天震荡换手，不再因为机械的“回落天数”直接踢掉。

这更接近你补充的审美：像绿色动力、兆新股份这种量能铺了十来天、5 日线踩得稳、涨得不算离谱的票，会比突然爆量加速的票优先级更高。

如果当天严格满足条件的股票不足 `5` 只，页面会继续先补“最接近趋势推进形态”的候选；如果仍不足，再补一层 `MA5` 更顺的备选形态，并在警告区明确标记。

此外，页面会基于当前筛选池，把最近 `10` 个可完整结算的信号日逐日回放一遍：信号日收盘选股、下一交易日开盘买入、第三交易日收盘卖出，并统计日均收益、累计收益、胜率以及每天的 5 只交易明细。

## 微信信号的工作方式

微信分析默认从下面这个目录读取导入文件：

```text
D:\message\data\wechat-import
```

支持的文件类型：

- `json`
- `jsonl`
- `ndjson`
- `txt`
- `md`
- `html`

系统会优先提取目标群聊和目标公众号的内容，并从最近 100 条群消息里重点识别：

- `顾子明`
- `子明群-子路`
- `间歇泉`

分析结果会输出：

- 当前判断：`偏科技向` / `偏战争线` / `拉锯中` / `信号不足`
- 科技线与战争线关键词得分
- 最近群消息摘录
- 最近公众号文章摘要
- 三位重点发言人的单独卡片

参考 [examples/wechat-import-sample.json](/D:/message/examples/wechat-import-sample.json)。

示例结构：

```json

  {

      }

  },
  {
    "sourceName": "子明解读",
    "articles": [
      {
        "timestamp": "2026-03-24T08:00:00+08:00",
        "title": "科技扩散与事件冲击之间",
        "content": "如果中东没有继续升级，市场主线仍在算力、芯片、机器人和国产替代。"
      }
    ]
  }
]
```

## 当前限制

当前这台机器已经检测到新版微信桌面端本地存储，但消息数据库不是明文 SQLite。也就是说：

- 这个版本不会直接解密读取微信实时消息库。
- 当前稳定方案是分析导入到 `data/wechat-import` 的可读文本或 JSON 文件。
- 只要导入文件更新，服务端就会在下一次 10 分钟轮询时自动重新分析。

## 说明

- 同花顺抓取逻辑依赖首页当前的 SSR 结构。
- 如果同花顺调整 DOM 或反爬策略，需要同步调整 [src/hotThemesService.js](/D:/message/src/hotThemesService.js)。
- 区间涨幅前十逻辑位于 [marketLeadersService.js](/D:/message/marketLeadersService.js)。
- 收盘趋势推进筛选逻辑位于 [patternSelectorService.js](/D:/message/patternSelectorService.js)。
- 如果后续拿到稳定的微信导出桥接程序，可以继续把第二部分升级成真正的自动同步。
